# CapyBoard — Demo 展示腳本

> 目標受眾：對桌遊設計有興趣的非技術用戶、投資人、合作夥伴  
> 時間：約 10～15 分鐘  
> 準備：開啟瀏覽器，確認已清除 localStorage（或使用無痕視窗）

---

## 第一幕：初次見面（1 分鐘）

**說話要點：**
> 「桌遊大師讓你不需要寫程式，就能設計並執行自己的桌遊規則。」

**操作：**
1. 開啟首頁 → 展示 4 個範例遊戲卡片
2. 指出「新手教學」按鈕，說明有完整引導
3. 點擊「使用說明」，快速看一下 Onboarding Tour（可跳過）

---

## 第二幕：從範例開始（3 分鐘）

**操作：**
1. 點擊「淘金熱」→「玩」
2. 進入遊戲執行頁面，說明介面：
   - 左側：玩家狀態（名字、分數、Token）
   - 中間：操作區、動作選擇
   - 右側：規則總覽（觸發時機、條件、動作）
3. 開始遊戲，執行幾個動作（採礦 → 冶煉 → 打磨寶石）
4. 展示事件日誌、回合歷史
5. 點擊「修改狀態」，手動加寶石到 3 個，觸發勝利條件

**說話要點：**
> 「這個遊戲用了資源鏈設計：礦石 → 黃金 → 寶石。贏得方式是收集 3 個寶石。
> 這些規則都是用我們的規則編輯器設計的，不需要任何程式碼。」

---

## 第三幕：進入編輯模式（4 分鐘）

**操作：**
1. 返回首頁 → 點擊「骰子冒險」→「編輯」
2. **元件 Tab**：展示 Token（帶 emoji 圖示）、拖放到棋盤
3. **動作 Tab**：點一個動作，說明動作類型和參數
4. **規則 Tab**：展示 `stamina_penalty` 規則（體力歸零落敗）
   - 指出 `hasVariable` 條件 → 連接到遊戲變數
5. **變數 Tab**：展示「體力」變數（預設 10）
6. 修改規則：把勝利分數改成 10（原本 15），儲存
7. 點擊「⚡ 快速測試」Drawer：不切分頁即可測試

**說話要點：**
> 「這個遊戲展示了新的『遊戲變數』功能。你可以定義血量、魔力、體力等自訂數值，
> 在規則中引用它。比如體力歸零就落敗，這讓遊戲設計更豐富。」

---

## 第四幕：進階功能亮點（3 分鐘）

### 4a. AND/OR 條件組合
**操作：**
1. 在規則編輯器 → 新增規則
2. 條件類型選「AND（所有條件成立）」
3. 加入兩個子條件：`hasTokenCount` + `hasScore`
4. 說明：「這讓勝利條件更精確，例如：同時擁有 3 個金幣 AND 分數 ≥ 5」

### 4b. 觸發鏈
**操作：**
1. 動作選「觸發另一條規則」
2. 選擇目標規則
3. 說明連鎖反應：一條規則觸發另一條規則

### 4c. Token 自訂圖示
**操作：**
1. 在遊戲編輯器點選一個 Token
2. 右側屬性面板：修改 Emoji 圖示
3. 即時看到 Token 卡片更新

---

## 第五幕：模組管理（2 分鐘）

**操作：**
1. 返回首頁
2. 展示「複製」功能：複製「卡牌大師」
3. 展示「匯出 JSON」：說明可分享給朋友
4. 展示「匯入 JSON」：上傳剛匯出的檔案

**說話要點：**
> 「遊戲以 JSON 格式儲存，可以分享、備份，也可以讓其他人繼續修改。
> 這是一個開放的格式，未來可以擴充到社群分享平台。」

---

## 第六幕：總結與展望（1 分鐘）

**已完成功能：**
- ✅ 遊戲編輯器（Token、動作、玩家、變數、棋盤）
- ✅ 規則編輯器（多種條件、AND/OR 組合、觸發鏈）
- ✅ 遊戲執行（完整回合、骰子動畫、勝利/落敗）
- ✅ 測試工具（快速測試、狀態倒退、修改狀態）
- ✅ 模組管理（新增/匯入/匯出/複製）
- ✅ 教學系統（新手引導、Tooltip、第一個遊戲教學）

**Roadmap（未來）：**

## 常見問題應對

| 問題 | 回答 |
|------|------|
| 「這能做多複雜的遊戲？」 | 目前支援資源管理、回合制、卡牌、骰子、棋盤移動等主要桌遊機制。複雜的連鎖規則可用觸發鏈實現。 |
| 「可以存檔嗎？」 | 自動存在瀏覽器 localStorage，也可匯出 JSON 永久保存。 |
| 「能多人聯機嗎？」 | 目前是單機版，需要玩家共享同一台電腦。多人連線是下一階段目標。 |
| 「可以自訂圖片嗎？」 | 目前支援 Emoji 圖示，圖片上傳功能在規劃中。 |
